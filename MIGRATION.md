# Repository Migration Plan

This document describes the migration of the API documentation from `tecafrik-git/truetms-api-docs` to `T3-Tech-Inc/truetms-api-docs`.

## Current State

- Documentation is generated by [SpectaQL](https://github.com/anvilco/spectaql) via live introspection of `beta-api.truetms.com`.
- It is served at [docs.truetms.com](https://docs.truetms.com) via GitHub Pages on the `tecafrik-git/truetms-api-docs` repository.
- The `T3-Tech-Inc/truetms-api-docs` repository holds the same content but does not have GitHub Pages configured yet.

## Migration Steps

### Phase 1 — Documentation update (complete)

1. Run SpectaQL to regenerate the documentation from the latest beta schema:
   ```bash
   npx spectaql config.yml
   ```
2. Commit and push the updated `docs/` directory to `tecafrik-git/truetms-api-docs` (main branch).
3. GitHub Pages automatically redeploys to [docs.truetms.com](https://docs.truetms.com).

### Phase 2 — Migration to T3-Tech-Inc (pending)

**Prerequisites:** Saalihou must grant admin access on `T3-Tech-Inc/truetms-api-docs` to proceed.

1. **Sync T3-Tech-Inc with the latest commit:**
   ```bash
   git remote add t3 https://github.com/T3-Tech-Inc/truetms-api-docs.git
   git push t3 main
   ```

2. **Activate GitHub Pages on T3-Tech-Inc** (requires admin):
   ```bash
   gh api repos/T3-Tech-Inc/truetms-api-docs/pages \
     -X POST \
     -f "source[branch]=main" \
     -f "source[path]=/docs"
   ```

3. **Validate the new deployment** — open `https://T3-Tech-Inc.github.io/truetms-api-docs` and confirm the documentation loads correctly.

4. **Set the custom domain on T3-Tech-Inc:**
   ```bash
   gh api repos/T3-Tech-Inc/truetms-api-docs/pages \
     -X PUT \
     -f "cname=docs.truetms.com"
   ```

5. **Update the DNS CNAME** (external action — requires DNS access):
   - Change `docs.truetms.com` CNAME from `tecafrik-git.github.io` → `T3-Tech-Inc.github.io`
   - Wait for DNS propagation (a few minutes to a few hours)
   - GitHub will automatically provision HTTPS for the new domain

6. **Disable GitHub Pages on tecafrik-git** (after full validation):
   ```bash
   gh auth switch --user nangui
   gh api repos/tecafrik-git/truetms-api-docs/pages -X DELETE
   ```

## Zero-Downtime Strategy

- Do **not** disable Pages on `tecafrik-git` until `T3-Tech-Inc` is fully validated and the DNS switch is confirmed.
- The DNS switch is the only moment where a brief propagation delay may occur — this is typically under a minute.
- `docs/CNAME` contains `docs.truetms.com` and must remain present in the repository throughout the migration.

## Key Files

| File | Purpose |
|---|---|
| `config.yml` | SpectaQL configuration — introspection URL, output directory, branding |
| `docs/` | Generated documentation (HTML + assets) — do not edit manually |
| `docs/CNAME` | Custom domain declaration — must not be removed |
| `content/authentication.md` | Manually authored content injected into the documentation |
| `package.json` | SpectaQL v2.3.1 as the sole dependency |

## Regenerating the Documentation

To update the documentation after API schema changes:

```bash
npm install        # only needed once
npx spectaql config.yml
git add docs/
git commit -m "docs: regenerate API documentation from beta schema"
git push origin main
```
